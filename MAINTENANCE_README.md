# 유지보수 문서

이 문서는 플러그인 구조를 수정하거나 유지보수하는 담당자를 위한 문서입니다.

## 실제 수정 포인트

- 시안 구조 및 생성 로직 수정: `code.js`
- UI 문구, 스타일, 입력 검증 수정: `ui.html`
- 플러그인 메타 정보 수정: `manifest.json`
- 구현 참고용 템플릿 구조 확인: `merged-react-figma.json`

## 파일 역할

- `manifest.json`: Figma 플러그인 설정 파일
- `code.js`: 플러그인 메인 실행 코드
- `ui.html`: 플러그인 UI 및 입력 검증 로직
- `merged-react-figma.json`: 이번 구현 기준 템플릿 구조 참고용 JSON
- `deploy.sh`: GitHub 배포 스크립트
- `gpush`: `deploy.sh` 단축 실행 파일

## 현재 구현 기준

- 4개 시안을 내부 데이터 기준으로 생성
- 각 시안은 독립 Frame으로 생성
- 선택 노드가 있으면 해당 위치 기준 배치
- 선택 노드가 없으면 현재 보이는 캔버스 영역 시작점 기준 배치
- 입력 레이어명 기준:
  - `job_title`
  - `speaker_name`
  - `d_day`
  - `img`

## 수정 시 유의사항

- 텍스트 반영 규칙과 이미지 반영 규칙은 `code.js`에서 함께 관리합니다.
- UI 검증 규칙은 `ui.html`에서 관리합니다.
- 사용자용 문구를 바꿀 때는 실제 플러그인 UI와 `README.md`를 같이 맞추는 것이 좋습니다.
- 구조 변경 시 `merged-react-figma.json`도 참고 파일 기준으로 함께 갱신하는 것이 좋습니다.

## 테스트 포인트

- 입력 필수값 동작 확인
- D-day 한 자리 숫자 입력 제한 확인
- 드라이브 링크와 업로드 파일 상호 배타 동작 확인
- 4개 시안이 가로 배치되는지 확인
- 선택 노드 기준 생성 / 미선택 시 기준 위치 생성 확인

## 운영 메모

- 현재 실행 파일은 `code.js`입니다.
- 템플릿 데이터는 실행 시 `code.js`와 `ui.html` 내부 값을 사용합니다.
- 구글 드라이브는 폴더 링크가 아니라 이미지 파일 1개의 공개 링크를 사용해야 합니다.
